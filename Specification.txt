◆概要
AWS Amplify を用いて簡易的なマルチユーザBlogアプリを作成する。
フロントエンドはReactを用いる。
CSSでのデザインは最低限度でよい。見た目は問わない。
ちゃんとシステムとして動作することを優先する。
※UIパーツで楽をする場合は、materialUI のライブラリを使用する。
　https://material-ui.com
　
◆AWSアカウント
　自分個人で、「使い捨て」のAWSアカウントを新規に作成して、
　その１年間無料枠で自作すること。
　※自分のクレジットカードで登録する。
　※課金されないように、課金サービスが無いか自分でチェックすること。
　※クレジットカードが無い場合は、社長か林に相談する。
　　AWS学習用アカウント
　
◆技術要素（必要となる技術）
　①react, materialUI
　②HTML, CSS
　③AWS Amplify シリーズ
　　AWS Amplify Console（アプリを公開）
　　AWS Amplify cli（バックエンドを構築）
　　AWS Amplify js（フロントエンドとバックエンドの連携）
　④AWS Appsync
　⑤AWS Cognito
　⑥AWS DynamoDB
　⑦AWS Lambda
　⑧AWS S3
　
◆機能要件（テスト時に会社側が評価としてチェックすること）
　ユーザはまずアカウントを作成し、Blog記事を新規作成する。
　Blog記事は他のユーザが閲覧、コメント、イイねができる。
　それを AWS Amplify を用いて一般公開できるようにすること。
　①認証機能（想定所要時間：４時間）
　　新規登録・ログイン・ログアウトができること。
　　ID,パスワードの入力間違いの場合は、エラーを表示すること。
　　プロフィール（ユーザー名とメールアドレス）が表示されること。
　②記事投稿機能（想定所要時間：4時間）
　　記事入力欄の内容が、フロントエンドに反映されること。
　　写真画像も記事と同じく登録できるようにする。
　　記事は、タイトル、本文、メイン画像、投稿者のページ構成（モデル）で作成する。
　　※本文中の画像は不要。
　③記事の一覧・閲覧機能（想定所要時間：３時間）
　　入力した記事の一覧（リスト）、
　　詳細ページ表示がそれぞれできること（第３者が閲覧できること）。
　④記事へのコメント機能（想定所要時間：３時間）
　　第３者が記事にコメントを記入でき、
　　それがフロントエンドに反映されること。
　⑤記事へのいいね機能（想定所要時間：４時間）
　　記事に対して第３者がイイねボタンを押すと、
　　その数がカウントされ、フロントエンドに表示されること。
